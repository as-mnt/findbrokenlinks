# Задача

## Исходная постановка

> Нужно спроектировать утилиту для поиска битых ссылок на сайте. На вход принимает url сайта,
> обходит его и фиксирует ссылки, результат открытия которых в браузере не соответствовал бы
> ожиданиям пользователя. Например, 404, редирект в странное место, сообщение CMS об
> отсутствующем файле. Язык для реализации пусть будет python, архитектура должна быть хорошо
> структурированной, контроли — в отдельных методах, чтобы можно было просто добавить новые
> контроли. При необходимости стоит применять распространённые паттерны. На выходе должен быть
> список, в котором должны быть битые ссылки, ссылки на разделы сайта, в которых размещены
> битые ссылки, и результаты обращения по битым ссылкам. Список нужно предоставлять в разных
> формах по выбору пользователя — csv, json, tabbed text, что ещё может пригодиться?

## Решения по уточняющим вопросам

| Вопрос | Решение |
|---|---|
| Какие ссылки проверять | Все три режима: `page` / `internal` / `internal+external`, переключаются ключом `--mode` |
| JS-рендеринг | Не нужен, только сырой HTML через httpx |
| Конкурентность | `asyncio` + `httpx` |
| Дополнительные форматы вывода | HTML, Markdown, JUnit XML, SARIF (+ обязательные csv/json/tsv) |
| robots.txt | Уважаем по умолчанию, отключается флагом `--ignore-robots`; rate-limit настраивается через `--rate-limit` |
| Soft-404 | Оба механизма: regex-паттерны + probing с эталонным 404 |
| Подозрительные редиректы | Редирект на `/` (home) + цепочка >N редиректов (N настраивается) |
| Что проверять помимо `<a href>` | `<img src>`, `<script src>`, `<link href>` |

## Что считается "битой ссылкой"

Любая ссылка, открытие которой в браузере не соответствовало бы ожиданиям пользователя:

- HTTP 4xx / 5xx (`HTTP_ERROR`)
- Сетевые сбои: timeout, DNS, SSL, отказ соединения (`NETWORK_ERROR`)
- Редирект на главную страницу (`REDIRECT_TO_HOME`) — типичный soft-404
- Длинная цепочка редиректов (`REDIRECT_CHAIN`)
- Страница вернула 200, но содержит признаки "не найдено": шаблонные тексты CMS (`SOFT_404_PATTERN`)
- Страница вернула 200, но её тело совпадает с baseline-404 домена (`SOFT_404_PROBE`)

## Что должно быть в отчёте

Для каждой битой ссылки:

- `source_page` — страница, где найдена ссылка
- `link_url` — целевой URL ссылки
- `final_url` — куда фактически попали после редиректов
- `anchor` — текст / alt ссылки
- `tag` — `a` / `img` / `script` / `link` / `seed`
- `status` — HTTP-код
- `redirect_chain` — список промежуточных URL
- `issue_codes` — какие контроли сработали
- `severity` — error / warning / info
- `details` — человекочитаемое объяснение
